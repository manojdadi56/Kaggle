[Discussion] Concerns about copied notebooks and misleading submissions in the Notebooks section , Heavy Plagiarism
I’d like to bring attention to a pattern I’ve been noticing recently in the Notebooks section that may be harmful to the integrity of the platform. After This Guy everyone has started to do the same following his path ; even worse and they are not getting banded or removed!!
There are several notebooks being published with high claimed scores, but upon closer inspection:
Minimal or no original work: Some notebooks appear to be direct copies of existing public notebooks, with little to no modification. Output reuse: In certain cases, the outputs or submission files seem to be reused rather than generated by the code in the notebook itself. Misleading titles: Titles sometimes claim scores or improvements that are not reproducible from the provided code. Limited transparency: Comments are occasionally disabled, which makes it harder for others to question or verify the work.
Here are a few examples for reference (for transparency and discussion purposes):
Notebook A → derived from Notebook B → originally from Notebook C Notebook D → very minimal changes from Notebook E
(I encourage others to verify these independently by comparing code and outputs.)
This kind of behavior can:
Mislead newer participants Reduce trust in shared work Devalue genuine contributions from the community
I’m not suggesting that reuse or learning from others is wrong—on the contrary, it’s a core part of Kaggle. But proper attribution, meaningful modification, and transparency are important.
Suggestions:
Encourage users to clearly state what is original vs reused Make it easier to report misleading notebooks Promote reproducibility (code should match results)
I’d be interested to hear how others view this and whether there are better ways to address it as a community.
