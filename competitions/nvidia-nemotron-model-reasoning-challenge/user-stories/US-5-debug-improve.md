# US-5 — Debug root-cause & improvement stories (reactive)

> As the system, when a run underperforms or fails, I want a root-cause task that proposes a concrete fix.

Created by the operator from a failed/weak `experiments/<id>` or from user feedback. Each is a scoped "diagnose X → propose/implement fix" Jules task. Independent issues run in parallel; one issue is serial. Template: `tasks/todo/TASK-5.x-<slug>.md` with the failing evidence + hypothesis + fix scope.
